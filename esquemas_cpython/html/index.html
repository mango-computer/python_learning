Nivel 1 — Script simple (un proceso, un hilo, sin async)

Un proceso, un hilo. Stack: un frame por llamada. Heap: objetos Python.

Nivel 2 — Asyncio (event loop, coroutines, un hilo)

Event loop en el hilo principal. Coroutines y Tasks en heap; el loop las programa y ejecuta cuando hacen await.

Nivel 3 — Thread pool (hilo principal + workers)

Un proceso, varios threads. Pool de workers con su propia cola. Heap compartido; un stack por thread.

flowchart LR subgraph PROC["Proceso Python"] subgraph MAIN["Hilo principal"] APP["Tu código - asyncio.run"] end subgraph POOL["Thread pool"] W1["Worker 1"] W2["Worker 2"] WN["Worker N"] end end subgraph MEM["Memoria"] HEAP["Heap compartido - cola pool, Futures"] S1["Stack main"] S2["Stack workers"] end APP --> HEAP POOL --> HEAP MAIN --> S1 POOL --> S2

Nivel 4 — Un proceso: threads + async en el main

Event loop y coroutines solo en el hilo principal. Thread pool ejecuta funciones síncronas; resultados vuelven vía Future.

flowchart TB subgraph PROC["Proceso"] subgraph MAIN["Hilo principal"] subgraph EL["Event loop"] C1["Coroutine 1"] C2["run_in_executor"] end end subgraph WORKERS["Thread pool"] W1["Worker 1"] W2["Worker 2"] end end C2 -->|submit fn| WORKERS WORKERS -->|Future result| EL HEAP["Heap: Futures, cola pool, Tasks"] MAIN --> HEAP WORKERS --> HEAP

Nivel 5 — Varios procesos (cada uno con threads y opcionalmente async)

Cada proceso tiene su propio heap y stacks. Comunicación por Queue/Pipe (serialización).

Demo: ¿Se bloquea el event loop? (ejemplo_bloqueo_vs_executor)

Se lanzan dos coroutines con gather(indicador_loop(5), tarea_...). indicador_loop imprime [loop] vivo cada 0,5 s. Si el loop está bloqueado, no imprime.

Parte 1 — Llamada directa (bloquea)

r = trabajo_cpu_corto(10, 2.0) dentro de la coroutine. Función síncrona: no hay await, el hilo se queda ~2 s en time.sleep(2). El event loop no puede ejecutar indicador_loop → no ves [loop] vivo durante esos 2 s.

flowchart LR subgraph Main["Hilo principal - BLOQUEADO"] T["tarea: trabajo_cpu_corto(10, 2)"] end I["indicador_loop: no corre"] T -.->|"2 s bloqueado"| I

Parte 2 — run_in_executor (no bloquea)

r = await loop.run_in_executor(None, trabajo_cpu_corto, 10, 2.0). La coroutine cede; trabajo_cpu_corto corre en un worker del pool. El event loop sigue ejecutando indicador_loop → sigues viendo [loop] vivo cada 0,5 s.

flowchart TB subgraph Main["Hilo principal"] CO["coroutine: await run_in_executor"] IND["indicador_loop: imprime cada 0.5 s"] end subgraph Pool["Thread pool"] W["Worker: trabajo_cpu_corto(10, 2)"] end CO -->|submit| Pool Main --> IND

Ejemplo 12_async_advanced: run_in_executor (get_running_loop + ThreadPool)

loop = asyncio.get_running_loop() → await loop.run_in_executor(None, trabajo_cpu, 100). La coroutine cede; un worker del pool ejecuta trabajo_cpu; el resultado vuelve por Future y el loop reanuda la coroutine.

Future.set_result"| EL EL -->|"3. reanuda coroutine"| CO
sequenceDiagram participant C as Coroutine participant L as Event loop participant P as Thread pool C->>L: await run_in_executor(None, trabajo_cpu, 100) L->>P: submit trabajo_cpu 100 C->>C: (cede, espera Future) P->>P: ejecuta trabajo_cpu P->>L: Future done, resultado 5050 L->>C: reanuda con 5050

Ejemplo 12_async_advanced: Producer-consumer (asyncio.Queue + gather)

cola = asyncio.Queue() → await asyncio.gather(productor(cola, 5), consumidor(cola, 1), consumidor(cola, 2)). Un productor y dos consumidores en el mismo event loop; la cola coordina sin bloquear el hilo.
